# PR 审查

请对 Pull Request 进行全面审查。

## 审查内容
1. **改动概述** — 这个 PR 做了什么，解决什么问题
2. **代码质量** — 逻辑正确性、代码风格、命名规范
3. **安全审查** — 认证授权、输入校验、敏感数据处理
4. **性能影响** — 是否引入性能问题
5. **测试覆盖** — 关键路径是否有测试
6. **向后兼容** — 是否有破坏性改动
7. **文档更新** — API 变更是否更新了文档

## 使用方式
- `/pr-review` — 审查当前分支相对 main 的改动
- `/pr-review <branch>` — 审查指定分支的改动
- `/pr-review <pr-number>` — 审查指定 PR（需要 gh CLI）

## 输出格式
### 📋 PR 概述
- 改动目的和范围

### ✅ 优点
- 做得好的地方

### ⚠️ 建议改进
- 按严重程度排列的问题和建议

### 🔄 必须修改
- 阻塞性问题，必须修复才能合并

### 💡 总结
- 是否建议合并的结论
