# 代码解释

请解释指定代码的功能和逻辑。

## 解释维度
1. **功能概述** — 这段代码做什么，解决什么问题
2. **执行流程** — 逐行/逐块分析执行逻辑
3. **关键概念** — 涉及的设计模式、算法、API 用法
4. **输入输出** — 参数含义、返回值、副作用
5. **关联关系** — 谁调用了它、它调用了谁、在整体架构中的位置
6. **注意事项** — 潜在的坑、边界情况、性能考量

## 使用方式
- `/explain` — 解释当前选中的代码或打开的文件
- `/explain auth middleware` — 解释认证中间件
- `/explain database schema` — 解释数据库设计

## 输出风格
- 先给一句话总结
- 再展开详细解释
- 适当使用类比帮助理解
- 标注关键代码行号
