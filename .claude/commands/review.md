# 代码审查

请审查当前 git 改动，从以下维度分析：

## 审查维度
1. **安全漏洞** — SQL注入、XSS、认证绕过、敏感信息泄露
2. **Bug 风险** — 逻辑错误、空指针、竞态条件、边界情况
3. **性能问题** — N+1查询、内存泄漏、不必要的重渲染、大数据处理
4. **代码规范** — 命名一致性、错误处理、重复代码、过长函数
5. **最佳实践** — 设计模式、可测试性、可维护性

## 输出格式
- 按严重程度排序：🔴 严重 → 🟠 高 → 🟡 中 → 🔵 低
- 每个问题给出：文件位置、问题描述、修复建议
- 最后给出总结和优先修复建议

请先运行 `git diff` 查看改动，然后开始审查。
