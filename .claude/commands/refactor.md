# 重构建议

请分析指定代码并给出重构建议。

## 分析维度
1. **代码异味** — 过长函数、过大类、重复代码、深层嵌套
2. **职责分离** — 单一职责原则、关注点分离
3. **可读性** — 命名改进、注释补充、逻辑简化
4. **可测试性** — 依赖注入、纯函数提取、副作用隔离
5. **设计模式** — 是否适合引入策略、工厂、观察者等模式

## 输出格式
### 当前问题
- 列出发现的代码异味和改进点

### 重构方案
- 每个方案包含：
  - 目标：要解决什么问题
  - 方法：具体怎么改
  - 代码示例：重构前后的对比
  - 影响范围：改动会影响哪些文件

### 优先级
- 按收益/风险排序，建议重构顺序

## 使用方式
- `/refactor` — 重构当前打开的文件
- `/refactor userController.js` — 重构指定文件
- `/refactor authentication flow` — 重构指定功能模块
