# 调试助手

请根据错误信息定位问题根因并给出修复方案。

## 调试步骤
1. **理解错误** — 解读错误信息、堆栈跟踪
2. **定位源头** — 找到出错的文件和行号
3. **分析原因** — 为什么会出现这个错误
4. **给出修复** — 提供具体的代码修改方案
5. **预防措施** — 如何避免类似问题再次发生

## 使用方式
- `/debug TypeError: Cannot read property 'id' of undefined` — 根据错误信息调试
- `/debug login API returns 401` — 描述问题现象
- `/debug` — 分析当前选中的代码或最近的错误日志

## 调试工具
- 检查相关文件的代码逻辑
- 搜索错误信息在代码库中的上下文
- 分析数据流和调用链
- 检查配置文件和环境变量

## 输出格式
1. 🔍 **错误解读** — 这个错误是什么意思
2. 📍 **问题定位** — 出错的具体位置
3. 🔧 **修复方案** — 具体的代码修改
4. 🛡️ **预防建议** — 如何避免再次发生
