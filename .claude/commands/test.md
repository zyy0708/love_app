# 生成测试

请为指定的文件或函数生成单元测试。

## 要求
1. 分析目标文件的导出函数和关键逻辑
2. 生成覆盖以下场景的测试用例：
   - 正常输入（happy path）
   - 边界条件（空值、极值、特殊字符）
   - 错误处理（异常输入、网络错误、权限不足）
3. 使用项目现有的测试框架（如无则推荐合适的）
4. 测试命名清晰，描述预期行为
5. 包含必要的 mock/stub

## 使用方式
- `/test` — 测试当前打开的文件
- `/test src/utils/auth.js` — 测试指定文件
- `/test generateToken` — 测试指定函数

请先读取目标文件，分析其功能，然后生成测试。
